Manejo de Colisiones en Tablas Hash

Post #2 — Estrategias para resolver colisiones

← Volver al inicio

¿Qué son las colisiones?

Una colisión ocurre cuando dos claves diferentes generan el mismo índice a través de la función hash. Esto es inevitable en tablas hash de tamaño finito, por lo que necesitamos estrategias para manejarlas sin perder datos.

Estrategias comunes

Aquí comparamos las dos técnicas más populares: Encadenamiento y Direccionamiento Abierto. Ambas mantienen la eficiencia promedio en O(1), pero difieren en implementación y uso de memoria.

1. Encadenamiento (Chaining)

  • Cada posición en el arreglo de la tabla hash almacena una lista enlazada (o array dinámico) de pares clave-valor.
  • Si hay colisión, simplemente se agrega el nuevo elemento a la lista en ese índice.
  • Ventajas: Fácil de implementar, crece dinámicamente sin necesidad de redimensionar el arreglo principal con frecuencia.
  • Desventajas: Puede usar más memoria si hay muchas colisiones, y en el peor caso (todas las claves en una sola lista), las operaciones se vuelven O(n).

2. Direccionamiento Abierto (Open Addressing)

  • No usa listas; en cambio, busca la siguiente posición disponible en el arreglo si hay colisión.
  • Métodos populares:
    • Linear Probing: Busca la siguiente celda libre de forma lineal (índice +1, +2, etc.).
    • Quadratic Probing: Usa incrementos cuadráticos (índice +1², +2², etc.) para reducir agrupamientos.
    • Double Hashing: Usa una segunda función hash para calcular el salto, ofreciendo mejor distribución.
  • Ventajas: Mejor uso de memoria (sin punteros extras), ideal para arreglos estáticos o entornos con restricciones de memoria.
  • Desventajas: Requiere manejar el "clustering" (agrupamientos), y la tabla puede llenarse rápidamente, necesitando redimensionamiento.

Comparación

Elige chaining si priorizas simplicidad y escalabilidad; open addressing si buscas alto rendimiento en memoria cache. En la práctica, lenguajes como Python usan chaining en sus dicts.
